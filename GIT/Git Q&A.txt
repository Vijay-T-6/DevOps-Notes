Q) Merged local to remote but not able to see changes in remote what could be the reason
1. Check whether the branch is correct or not
check where did u push(git remote show origin)

2. Remote Branch Not Updated
If you pushed to a new branch, it might not be visible on the remote until you create a pull request or switch to it manually.

3. Push Didn't Actually Happen
Maybe the push failed silently or was interrupted.
Check your push history:

git log -n 1
git status

5. Pushed to a Fork or Different Remote
If you have multiple remotes (e.g., origin, upstream), you might have pushed to the wrong one.
Check remotes:
git remote -v

Detached HEAD or Rebase Confusion
If you were in a detached HEAD state or rebased and didn't push forcefully, the changes might not have gone through.
